Invokes a script closure with the given arguments.

This function handles both:

  • ScriptFunctionType (AST interpreter) - calls via its call method
  • ObjClosure (VM) - calls via the VM's callClosure method

The interpreter parameter should be either an Interpreter (AST) or VM (bytecode). The closure should be either a ScriptFunctionType or ObjClosure.

Throws a RuntimeError if the closure type doesn't match the interpreter type, or if the provided object is not a recognized closure type.

Implementation

Future<Object?> invokeScriptClosure(dynamic interpreter, ObjClosure closure, List<dynamic> arguments) async {
  // VM closure - interpreter must be a VM
  if (interpreter is! VM) {
    throw RuntimeError("ObjClosure requires a VM interpreter to invoke, but got ${interpreter.runtimeType}");
  }
  return await interpreter.callClosure(closure, arguments);
}
